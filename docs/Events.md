# Goal
現在の Events セクションは「3カラムのカード一覧」になっています。
これを Figma デザインのような「横スライダー（carousel）」に変更してください。

完成イメージは以下です。

- カードが横一列に並ぶ
- 初期表示の時点で、右端の次カードが少し見切れる
- 上部の左右ボタンで横にスライドする
- 各カードは「大きめのアーチ画像 + 下に日付、タイトル、本文、READ MORE」の構成
- 今のような強いカード枠・背景ボックス感は弱める
- 右端に見切れている画像は、デザインカンプから取得できる前提で使ってよい

# Current Problem
現状は以下の状態です。

- カードが3列グリッドで並んでいる
- 各カードにボックス感がある
- 右端見切れがない
- 「一覧UI」になっていて、Figma の「カルーセルUI」と骨格が違う

つまり、今回必要なのは小手先の余白調整ではなく、
**Events セクションの骨格を grid 一覧から carousel に変えること** です。

# Required Visual Changes
必ず以下を満たしてください。

## 1. レイアウト方式
- 現在の `display: grid` ベースの一覧レイアウトをやめる
- 外側に表示窓を作る
- 内側に横並びトラックを作る
- カードを固定幅で横一列に並べる
- 表示窓は `overflow: hidden` を使って右端カードを見切れさせる

## 2. カードの見た目
- カード全体の背景ボックス感を弱める
- 不要に強い背景色、枠線、シャドウ、過剰な padding があれば整理する
- 画像を主役にする
- 画像はアーチ型（上部丸み）で見せる
- テキストは画像下に配置する

## 3. 初期表示
- 初期状態で右端の次カードが少しだけ見える
- 「続きがある」と分かる見え方にする

## 4. ボタン挙動
- Prev / Next ボタンで横移動する
- アニメーションは自然で短め
- レイアウトシフトを起こさない
- リサイズ時も破綻しにくい

# Implementation Direction
実装方針は以下を優先してください。

## Layout
- `.events-slider` のような表示窓を作る
- `.events-track` のような横並びトラックを作る
- `display: flex` + `gap`
- カードは `flex: 0 0 <fixed-width>`
- 表示窓は `overflow: hidden`

## Motion
- JSで現在位置を管理
- `transform: translateX(...)` でトラックを動かす
- `transition` でスムーズにする

## Card appearance
- 画像リンクをブロック化
- アーチ形状は `border-radius` で作る
- 画像は `object-fit: cover`
- カード全体に強いボックス背景を置かない

# Constraints
