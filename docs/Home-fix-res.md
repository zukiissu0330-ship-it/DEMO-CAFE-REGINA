さっきの診断結果に基づいて、CAFE REGINA の HOME 画面をレスポンシブ修正してください。

# 目的
PCデザインの雰囲気を保ちつつ、
- タブレット
- スマホ
で大崩れしないように改善してください。

# 最優先
1. PC表示を壊さない
2. Heroの破綻を直す
3. Header / Newsletter / Footer の可読性を確保
4. 各セクションが横にはみ出さない
5. 無理に完全一致を狙わず、自然に縮退させる

# 実装ルール
- 既存コードを活かして修正する
- HOME以外は触らない
- できるだけ CSS中心で直す
- 不要なJS追加はしない
- 必要なら media query を追加する
- 色・フォントトークンは既存の tokens 定義を使う
- 変更箇所には分かりやすいコメントを残す

# レスポンシブ方針
## Header
- 狭い幅では nav の gap / font-size / padding を調整
- 必要なら 768px 以下で折り返し or 縦積みも可
- ただし初回はハンバーガー化しなくてよい

## Hero
- 巨大見出しは clamp() で縮小
- メイン画像は width / max-width / aspect-ratio を整理
- 左下 / 右上の装飾画像は画面幅に応じてサイズ縮小
- 極端に狭い画面では重なりを弱めてもよい
- pagination と CTA の位置は relative に近づけて安全化してよい

## Drinks / Small Hunger
- 3カラム固定で無理なら 1024以下で2段、768以下で縦積みにしてよい
- 本文の max-width を設定
- 中央画像がつぶれないようにする

## Weekend Suggestion
- 画像コラージュは狭い画面では重なりを減らしてよい
- 完全な斜め配置維持が厳しい場合は簡略化してもよい
- ただし雰囲気は残す

## Events
- PCでは横並び維持
- タブレット以下ではカード幅調整
- 必要なら横スクロールコンテナ化してもよい
- ただし見た目が破綻しないこと

## Photo Gallery
- 画像群は幅に応じて縮小
- 狭い画面では縦積み or 2列化可

## Newsletter
- input と button は狭い画面で縦積み可
- 文字サイズと余白を clamp() で調整

## Footer
- 3カラム → 2カラム → 1カラム の順で自然に落とす
- テキストは中央寄せでも可
- 住所や営業時間の可読性を優先

# 推奨ブレイクポイント
- 1200px
- 1024px
- 768px
- 480px

# clamp() 化を優先したいもの
- Hero巨大見出し
- セクション見出し
- 本文
- CTA文字
- セクション上下余白
- 画像サイズの一部

# pxのまま残してよいもの

- 円アイコンの線幅
- ボーダー幅
- 小さな装飾差分
