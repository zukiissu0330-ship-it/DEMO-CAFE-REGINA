# 目的
背景崩れを直してください。

## 直したいこと
1. 背景画像や背景色が下方向で途切れず、白い余白が見えないこと
2. Hero の拡縮時に、下の余白が不自然に増減しないこと
3. ページ全体が黒背景の世界観で自然につながること
4. 既存レイアウトをできるだけ壊さず、最小差分で直すこと

# 修正ルール
- `style.css` には通常CSSを書かない
- `css/pages/home.css` を中心に必要最小限だけ編集
- 必要なら `components.css` や HTML も最小限で修正してよい
- **absolute の乱用でごまかさない**
- 高さ不足の根本原因を直す
- 背景は「どの要素が持つべきか」を整理して実装する
- Hero の高さは、固定値ベタ打ちではなく、縮尺変化に耐えやすい形にする
- 1280px前後でも極端に崩れないようにする

# 改善の方向性
以下のどれか、または組み合わせで対応してください。
- Hero の親要素に適切な `min-height` / `padding-bottom` / `overflow:hidden` を与える
- absolute配置された要素によって親高さが消えているなら構造修正する
- 背景画像なら `background-size: cover` / `background-position` を適切に設定する
- 必要なら背景用の擬似要素や専用ラッパーを使う
- セクション切れ目で背景が破綻するなら wrapper 単位で統合する
